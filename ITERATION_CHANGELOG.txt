3. SMS ALERTS NOT SENDING TO DOCTOR
   ─────────────────────────────────
   Problem: SMS alerts were only logged, not actually sent.
            No Twilio integration was active.
   
   Solution: Enhanced Twilio SMS Implementation
   
   Changes Made:
   
   a) Dashboard (app/templates/dashboard.html, Line 302):
      • Updated sendSMS() to call /api/patient/{id}/alert
      • Now sends actual API request when alert triggered
      • Includes error handling and logging
   
   b) Backend (app/services/sepsis_engine.py, Line 269):
      • Enhanced send_alert() to check for Twilio client
      • Attempts real SMS send to doctor phone
      • Fallback to logging if SMS not configured
      • Returns status: "sent" or "logged"
   
   To Enable Real SMS:
   ──────────────────
   1. Create Twilio account at https://www.twilio.com
   2. Get credentials:
      - Account SID
      - Auth Token  
      - Twilio phone number (e.g., +1XXXXXXXXXX)
   
   3. Set environment variables (before running Flask):
      set TWILIO_ACCOUNT_SID=<your_account_sid>
      set TWILIO_AUTH_TOKEN=<your_auth_token>
      set TWILIO_PHONE_NUMBER=<your_twilio_number>
   
   4. Restart Flask server:
      python run.py
   
   5. SMS will auto-send to doctor phone numbers (e.g., 7339300849)
      when patient risk ≥ 75%, every 5 minutes
